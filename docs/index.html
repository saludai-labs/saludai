How it works

The Agent Loop

A plan-and-execute loop with a FHIR knowledge graph. The planner classifies the question and selects a strategy before the agent starts calling tools — every step traced in Langfuse.

1. Query 2. Plan 3. LLM Reasons 4. Resolve Terms 5. Search / Compute 6. Answer
System design

Architecture

Four packages, multiple entry points, full observability. Connect via MCP, REST API, or CLI.

Capabilities

Built for Precision

🏥

Medical Terminology

Fuzzy matching across 4 systems: SNOMED CT (Argentine edition), CIE-10, LOINC, and ATC. Resolves natural language to clinical codes with disambiguation via rapidfuzz.

🧠

Hybrid Query Planner

Plan-and-execute pattern with a FHIR knowledge graph (10 resource relationships, 11 query patterns). Classifies questions and selects strategy before the agent acts.

🔍

Full Observability

Every LLM call, tool execution, and FHIR query traced through Langfuse. Query plan, token usage, and cost per step. Clinical auditability by design.

📊

100-Question Benchmark

5 LLMs compared on 100 clinical questions across 10 FHIR resource types. Hybrid judge (programmatic + LLM). 17 experiments documented with per-question analysis.

🌎

Locale Packs

Country-specific bundles — terminology, prompts, FHIR metadata, search params. Argentina built-in (openRSD-aware). Designed to extend to other countries via entry_points.

🤖

Provider Agnostic

Same agent loop works with Claude, GPT-4o, Llama, Qwen, or any OpenAI-compatible API. Swap the model, keep everything else. MCP, REST API, and CLI entry points.

Get started

Up and Running in 4 Commands

terminal
# Clone and install
$ git clone https://github.com/saludai-labs/saludai.git && cd saludai
$ uv sync
 
# Start FHIR server with 200 synthetic Argentine patients
$ docker compose up -d
 
# Query (200 patients, 3,182 resources, 10 types)
$ uv run saludai query "Pacientes con diabetes tipo 2 mayores de 60"
