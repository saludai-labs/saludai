SaludAI

A benchmarked, traceable FHIR agent for Latin America — measurable precision across 5 LLMs, every step auditable.

saludai query
Designed to scale

Built for Argentina. Ready for Latin America.

SaludAI uses locale packs — pluggable bundles that contain a region's medical terminology, clinical prompts, and FHIR metadata. Switch the locale pack, and the entire agent adapts: codes, language, search strategies, even the evaluation benchmark.

🇦🇷

Argentina (built-in)

SNOMED CT Argentine edition, CIE-10, LOINC, ATC. openRSD-aware FHIR profiles. Spanish-first prompts. 200 synthetic patients with real Argentine demographics.

📦

Fully extensible

Add a locale pack for your country, province, or institution — national terminology, local FHIR profiles, and clinical vocabulary. No changes to the agent core.

🔄

Any FHIR R4 server

Not locked to a specific vendor. Connect to HAPI FHIR, Firely, Azure FHIR, Google Cloud Healthcare, or any R4-compliant server.

Evaluation

Multi-LLM Benchmark

100 clinical questions across 200 synthetic Argentine patients (3,182 FHIR resources, 10 types). Evaluated on deterministic seed data, not real clinical data. Inspired by FHIR-AgentBench. Hybrid judge (programmatic + LLM).

How it works

The Agent Loop

A plan-and-execute loop with a FHIR knowledge graph. The planner classifies the question and selects a strategy before the agent starts calling tools — every step traced in Langfuse.

System design

Architecture

Four packages, multiple entry points, full observability. Connect via MCP, REST API, or CLI.

Capabilities

Built for Precision

🏥

Medical Terminology

Fuzzy matching across 4 systems: SNOMED CT (Argentine edition), CIE-10, LOINC, and ATC. Resolves natural language to clinical codes with disambiguation via rapidfuzz.

🧠

Hybrid Query Planner

Plan-and-execute pattern with a FHIR knowledge graph (10 resource relationships, 11 query patterns). Classifies questions and selects strategy before the agent acts.

🔍

Full Observability

Every LLM call, tool execution, and FHIR query traced through Langfuse. Query plan, token usage, and cost per step. Clinical auditability by design.

📊

Benchmarked

100 questions across 10 FHIR resource types — simple, medium, and complex — following the FHIR-AgentBench taxonomy. Hybrid judge (programmatic + LLM). 5 models compared.

🤖

Provider Agnostic

Same agent loop works with Claude, GPT-4o, Llama, Qwen, or any OpenAI-compatible API. Swap the model, keep everything else. MCP, REST API, and CLI entry points.

Get started

Up and Running in 4 Commands

terminal
# Clone and install
$ git clone https://github.com/saludai-labs/saludai.git && cd saludai
$ uv sync
 
# Start FHIR server with 200 synthetic Argentine patients
$ docker compose up -d
 
# Query (200 patients, 3,182 resources, 10 types)
$ uv run saludai query "Pacientes con diabetes tipo 2 mayores de 60"
